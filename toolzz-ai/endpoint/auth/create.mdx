<Expandable title="Payload">
<ResponseField name="name" type="string" required>
  First name of the user.
</ResponseField>
<ResponseField name="lastName" type="string" required>
  Last name of the user.
</ResponseField>
<ResponseField name="email" type="string" required>
  Email address of the user.
</ResponseField>
<ResponseField name="password" type="string" required>
  Password for the user account.
</ResponseField>
<ResponseField name="password_confirmation" type="string" required>
  Confirmation of the password for verification.
</ResponseField>
</Expandable>

## Response Description 
Below you will see the description of the bot response

<ResponseField name="id" type="uuid" required>
  Unique identifier of the user.
</ResponseField>
<ResponseField name="name" type="string" required>
  First name of the user.
</ResponseField>
<ResponseField name="lastName" type="string" required>
  Last name of the user.
</ResponseField>
<ResponseField name="email" type="string" required>
  Email address of the user.
</ResponseField>
<ResponseField name="token" type="string" required>
  Authentication token for the user session.
</ResponseField>
<ResponseField name="institutionId" type="uuid">
  Identifier of the associated institution (optional).
</ResponseField>
<ResponseField name="institutionName" type="string">
  Name of the associated institution (optional).
</ResponseField>
<ResponseField name="isActive" type="boolean">
  Indicates whether the user account is active (optional).
</ResponseField>
<ResponseField name="hasPassword" type="boolean" required>
  Indicates whether the user has a password set.
</ResponseField>
<ResponseField name="superAdmin" type="boolean" required>
  Indicates whether the user has super admin privileges.
</ResponseField>
<ResponseField name="members" type="array" required>
  List of associated members.
</ResponseField>
<ResponseField name="isMember" type="boolean">
  Indicates whether the user is a member (optional).
</ResponseField>

## Segurança
Para acessar este endpoint, é necessário enviar um **token de acesso** válido através do cabeçalho de autorização (Authorization) da requisição. Além disso, a API é protegida por outras medidas de segurança para proteger os dados dos usuários.
