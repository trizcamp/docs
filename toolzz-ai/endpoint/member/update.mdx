---
title: 'Update Member'
description: Endpoint to Update a Member
api: 'PUT https://chat.api.toolzz.com.br/api/v2/member/id/{id}'
authorization: none
---

<RequestExample>
```json Payload Example
  {
    "access": "ADMIN"
    "name" : "<string>"
    "unityId": "<uuid>"
  }
```
</RequestExample>

## Response Description 
Below you will see the description of the member response

<ResponseField name="message" type="string" required>
  Message indicating the result of the operation.
</ResponseField>
<ResponseField name="id" type="string" required>
  Unique identifier of the member.
</ResponseField>
<ResponseField name="institutionId" type="string" required>
  ID of the associated institution.
</ResponseField>
<ResponseField name="unityId" type="string" required>
  ID of the associated unity.
</ResponseField>
<ResponseField name="userId" type="string" required>
  ID of the associated user.
</ResponseField>
<ResponseField name="name" type="string" required>
  Name of the member.
</ResponseField>
<ResponseField name="email" type="string" required>
  Email address of the member.
</ResponseField>
<ResponseField name="access" type="string" required>
  Access level of the member.
  <Expandable title="Allowed values">
    <ResponseField name="MANAGER" type="string">
      Access level for managing operations.
    </ResponseField>
    <ResponseField name="ADMIN" type="string">
      Access level for administrative control.
    </ResponseField>
    <ResponseField name="USER" type="string">
      Access level for general users.
    </ResponseField>
  </Expandable>
</ResponseField>
<ResponseField name="status" type="number" required>
  Status indicator, 1 for active, 0 for inactive.
</ResponseField>
<ResponseField name="acceptedAt" type="string" required>
  Timestamp indicating when the member was accepted.
</ResponseField>
<ResponseField name="createdAt" type="string" required>
  Timestamp indicating when the record was created.
</ResponseField>
<ResponseField name="updatedAt" type="string" required>
  Timestamp indicating when the record was last updated.
</ResponseField>
<ResponseField name="user" type="object" required>
  Details of the associated user.
  <Expandable title="User properties">
    <ResponseField name="id" type="string" required>
      Unique identifier of the user.
    </ResponseField>
    <ResponseField name="name" type="string" required>
      First name of the user.
    </ResponseField>
    <ResponseField name="lastName" type="string" required>
      Last name of the user.
    </ResponseField>
    <ResponseField name="email" type="string" required>
      Email address of the user.
    </ResponseField>
    <ResponseField name="phone" type="string">
      Optional phone number of the user.
    </ResponseField>
    <ResponseField name="password" type="string" required>
      Hashed password of the user.
    </ResponseField>
    <ResponseField name="imgProfileId" type="string">
      Optional profile image ID.
    </ResponseField>
    <ResponseField name="rememberMeToken" type="string">
      Optional remember me token.
    </ResponseField>
    <ResponseField name="verified" type="boolean" required>
      Indicates if the user's email is verified.
    </ResponseField>
    <ResponseField name="isMember" type="boolean" required>
      Indicates if the user is a member.
    </ResponseField>
    <ResponseField name="institutionId" type="string" required>
      Institution ID associated with the user.
    </ResponseField>
  </Expandable>
</ResponseField>

## Segurança
Para acessar este endpoint, é necessário enviar um **token de acesso** válido através do cabeçalho de autorização (Authorization) da requisição. Além disso, a API é protegida por outras medidas de segurança para proteger os dados dos usuários.
